# Payer: SME Financial Command Center & E-commerce Platform

**Payer** is a unified e-commerce and treasury management solution designed specifically for small and medium-sized enterprises (SMEs). It bridges the gap between sales and operations by providing a seamless shopping experience for customers while empowering business owners with automated tools for payroll and utility management within a single protected dashboard.

## Project Description
Payer is built to solve the fragmentation of financial tools in small businesses. Instead of using separate platforms for revenue collection and expense management, Payer consolidates these into one ecosystem. 

*   **For Customers:** It offers a modern, no-auth-required e-commerce storefront where users can browse products, manage a shopping cart, and pay via **Visa, Verve, Mastercard, bank transfers, or USSD** through a secure checkout experience.
*   **For Business Owners:** A protected admin side acts as a treasury "command center." Owners can monitor real-time sales, trigger **instant staff payroll** via bank transfers, pay for **office utilities (electricity/cable)**, and automate **staff data/airtime benefits** directly from the revenue earned.

## ✨ Key Frontend Features
*   **Guest Checkout:** Customers can view products and complete purchases without the friction of creating an account.
*   **Real-Time Dashboard:** The admin panel displays live financial data using **Nomba's Transactions API**, providing visibility into both incoming revenue and outgoing business expenses.
*   **Automated Staff Payroll Interface:** A dedicated section for managing staff records and initiating bulk transfers to bank accounts.
*   **Utility & Benefit Vending:** Built-in interfaces for selecting electricity providers or telcos to pay bills or vend data bundles.

## 🛠 Tech Stack & Integration
The frontend is designed to be highly responsive and secure, utilizing the following resources:
*   **Nomba React SDK:** Used to implement a seamless **Online Checkout** experience for customers.
*   **Transactions & Requery:** Interfaces built to fetch and display transaction histories and verify successful payments.
